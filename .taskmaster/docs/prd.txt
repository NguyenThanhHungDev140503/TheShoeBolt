# Product Requirements Document: TheShoe - Website Thương mại điện tử Bán Giày

## 1. Giới thiệu (Introduction)

### 1.1. Mục đích (Purpose)
Tài liệu này xác định các yêu cầu kỹ thuật và nghiệp vụ cho dự án phát triển backend (BE) của website thương mại điện tử TheShoe. Hệ thống BE sẽ cung cấp nền tảng vững chắc cho các chức năng của người dùng, quản trị viên và shipper, đồng thời đảm bảo hiệu suất, bảo mật và khả năng mở rộng.

### 1.2. Phạm vi (Scope)
Hệ thống BE sẽ bao gồm các module chính sau:
- Quản lý Người dùng (User Management)
- Quản lý Sản phẩm (Product Management)
- Quản lý Giỏ hàng (Cart Management)
- Quản lý Đơn hàng (Order Management)
- Quản lý Thanh toán (Checkout Management)
- Quản lý Khuyến mãi (Promotion Management)
- Quản lý Thông báo (Notification Management)
- Quản lý Wishlist (Wishlist Management)
- Quản lý Phản hồi (Feedback Management)
- Thống kê (Analytics)
- Quản lý Bộ sưu tập (Collection Management)
- Xác thực (Authentication via Clerk)
- Tích hợp Vận chuyển (Shipper Integration)
- Tích hợp Cổng thanh toán (Stripe Payment Gateway)
- Các module hạ tầng: RBAC, Xử lý lỗi, Cơ sở dữ liệu, Email, Logging, Cache, Lưu trữ tệp, Tìm kiếm, Hàng đợi tin nhắn, Cổng API, Xử lý Webhook.

## 2. Đối tượng người dùng (User Characteristics)

- **Khách hàng**: Người dùng cuối, duyệt sản phẩm, đặt hàng, thanh toán, quản lý tài khoản cá nhân.
- **Quản trị viên**: Quản lý hệ thống, sản phẩm, đơn hàng, người dùng, khuyến mãi, xem thống kê.
- **Shipper**: Thực hiện giao hàng, cập nhật trạng thái đơn hàng.

## 4. Yêu cầu phi chức năng (Non-Functional Requirements)

- **NFR-1**: Giao diện thân thiện, dễ sử dụng (Áp dụng cho cả trang người dùng và trang quản trị).
- **NFR-2**: Hiệu suất hệ thống:
    - Đáp ứng tối thiểu 100 người dùng đồng thời.
    - Thời gian phản hồi của API dưới 200ms cho 95% request.
    - Tối ưu hóa truy vấn CSDL, sử dụng indexing và prepared statements hiệu quả.
- **NFR-3**: Bảo mật:
    - Dữ liệu người dùng, đơn hàng, thanh toán phải được mã hóa.
    - Tuân thủ OWASP Top 10.
    - Sử dụng Clerk cho xác thực để tăng cường bảo mật.
    - Bảo vệ chống lại các tấn công phổ biến (SQL Injection, XSS, CSRF).
- **NFR-4**: Thời gian phản hồi trang web dưới 2 giây (Liên quan đến BE API performance).
- **NFR-5**: Hỗ trợ đa ngôn ngữ (Tiếng Anh, Tiếng Việt) cho dữ liệu sản phẩm, danh mục.
- **Khả năng mở rộng**: Hệ thống có khả năng mở rộng để đáp ứng lượng truy cập tăng cao.
- **Khả năng bảo trì**: Mã nguồn được tổ chức rõ ràng, dễ hiểu, dễ bảo trì và nâng cấp.
- **Độ tin cậy**: Hệ thống hoạt động ổn định, giảm thiểu thời gian downtime.

## 5. Thiết kế hệ thống tổng quan

### 5.1. Kiến trúc
- Mô hình Microservices hoặc Monolithic tùy theo quyết định cuối cùng (hiện tại theo hướng Monolithic với NestJS modules).
- Backend REST API sử dụng NestJS framework.
- Cơ sở dữ liệu: PostgreSQL.
- Xác thực: Clerk.
- Thanh toán: Stripe.
- Gửi email: Resend.
- Tìm kiếm: Elasticsearch (cân nhắc).
- Cache: Redis (cân nhắc).
- Hàng đợi tin nhắn: RabbitMQ/Kafka (cân nhắc cho các tác vụ bất đồng bộ).

### 5.2. Cấu trúc dữ liệu (Tham khảo ERD)
(Chi tiết trong file "Phân Tích ERD Hệ Thống Web Bán Giày (Report).pdf")
Các bảng chính: User, Product, Category, Order, OrderDetail, Cart, CartItem, Address, Payment, Shipping, DiscountCode, Promotion, Review, Wishlist, ProductImage, Collection, Role, Permission, và các bảng trung gian.
